portfolio website
- case study -

Context and challenge

Purpose

Create a portfolio website with HTML5, CSS3 and JavaScript to have a place to showcase the different projects I work on as a UX Designer/Front-end Developer. 

Role

I was in charge of producing the content of the site and writing the code for the site itself with HTML5, CSS3 and JavaScript. After creating the site I tested it with five users and implemented changes according to user feedback.

Timeline

09/19-11/19

Tools

Problem

I want to work as a UX Designer/ Front-end developer and therefore need an online presence to showcase my projects. This way, I can present my work to my future employers, potential clients and coworkers. Since it’s very likely that I will be judged by the quality of the website, it had to be of high quality. It had to follow industry-standard systems, using responsive media queries, implementation of a CSS-stylesheet and JavaScript libraries. It had to pass W3C validation and work in all modern browsers, in all viewport sizes and with keyboard, mouse and touch inputs. 

Goal

Build a site with the above objectives and gain online traffic to the site.

Process and Insights

Research

As first step I conducted a research on other professional UX portfolios to see how other designers present their work and content and which best practices and conventions would be appropriate to showcase the projects I worked on. Then I created a table of content with all the projects I wanted to show. These would not only be UX-related. To show the full scope of my design capabilities, I decided to also show pieces from my previous experiences as a textile designer and artist.

Design

The whole project was targeted with a mobile-first approach using media queries to define the breakpoints. To structure the content across different viewports (mobile, tablet and desktop), I laid out wireframes for every screen size I wanted to support. This way I could easily organize the site content into rows/columns, determine image sizes, text copy, etc. I developed a visual style beforehand: define the colors I wanted to use, the styling of buttons, fonts, etc. Then I started to write out the basic structure in an HTML-document and defined all styles in a CSS stylesheet to ensure consistent formatting of all visual elements. To enhance the user experience and provide more comfort in using my site I added interactive elements with JavaScript. For example a scroll-to-top button makes it easier to browse loads of content and interactive buttons provoke a CTA.

Testing

To discover possible errors in navigation and content, I conducted a user test with five participants. I developed a testing protocol with four scenarios and open-ended questions. This way I was able to detect some misleading and incomplete information such as a missing full name and missing underlining to help declare links. All errors were rated according to Jakob Nielsens severity rating scale. A list of recommendations for future iteration was compiled and will be worked off step by step

Solution

By employing industry standards from basic site structure in my HTML document to using code linters for debugging the code, I set up an responsive webapplication that demonstrates my design skills as well as my knowledge of coding a website. The site meets AA accessability standards, can be viewed in all major browsers (Chrome, Opera, Safari, Firefoy & Edge) and is optimized for use on all common device sizes. My own portfolio site can be used by fellow designers, hiring managers and new network contacts to learn about me and my work.
